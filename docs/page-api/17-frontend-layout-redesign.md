# 17. 前端排版重设计需求

- 状态：待用户试用后提交具体反馈
- 优先级：P1（影响整体使用体验）

## 已知问题

### 整体布局

- 页面整体偏紧凑，信息密度过高，缺少留白和呼吸感
- 多处区块之间间距不统一，视觉节奏混乱
- sidebar 和 main 区域比例在不同页面不一致

### 重复和冗余

- ExamShellView sidebar 和 main 区域存在信息重复（进度、保存状态在两边都显示）
- CandidateDetailView 三栏面板信息有交叉（review_notes 和 analysis.risks 语义相近）
- TaskDetailView 的"处理进度"和"上传列表"里进度信息重复展示

### PDF 预览

- CandidateDetailView 已有 PDF 预览组件（vue-pdf-embed），但在现有三栏布局中不够突出
- 建议：PDF 原件预览应该和解析结果并排对照，方便 HR 验证解析质量
- 可能需要重新规划详情页为"原件 | 画像 | 操作"三段式或 tab 切换

### 考试页

- 考生答题时 sidebar 信息量仍然过大（导航 + 计时 + 进度 + 保存状态 + 提交按钮全挤在一起）
- 建议：sidebar 只保留计时 + 题目导航 + 交卷按钮，其他信息折叠或移除
- 代码题编辑器和普通题之间切换时视觉跳跃明显

### 管理端

- Dashboard / TaskList / CandidateList 这些列表页风格接近但细节实现不一致（有的用 el-table，有的用自定义 card grid）
- 统一设计语言：表格页用 el-table，详情页用卡片布局
